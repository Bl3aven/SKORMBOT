# SKORMBOT - Règles de Travail

## ⛔ Permissions Discord - Ne pas modifier

**RÈGLE : Ne jamais modifier les permissions (overwrites) des salons Discord du serveur SKORM sans demande explicite de l'utilisateur.**

### Contexte
- Le serveur Discord a une structure de permissions / rôles / salons déjà configurée.
- Les modifications de permissions peuvent casser l'accès des membres, briser les catégories, ou causer des problèmes de vérification.
- Le bot a déjà les permissions nécessaires via ses rôles.

### Ce qui est INTERDIT sans demande explicite
- Modifier les overwrites d'un salon (`channel.set_permissions()`, `channel.edit(overwrites=...)`)
- Créer ou supprimer des overwrites pour un rôle ou un membre sur un salon
- Changer les permissions de vérification sur les salons
- Réinitialiser les permissions d'un salon ou d'une catégorie
- Syncer les permissions des salons (ex: après création de salons)

### Ce qui est AUTORISÉ
- Lire les permissions existantes pour diagnostiquer
- Modifier le code du bot lui-même (cogs, commandes, événements)
- Modifier la configuration Docker, Lavalink, etc.
- Ajouter des fonctionnalités qui n'impliquent pas de changement de permissions

### Exception
Si l'utilisateur demande explicitement de modifier les permissions d'un salon, c'est autorisé. Dans ce cas, demander confirmation avant d'appliquer les changements.
